DAY 6 Phase 2 — System API

PostgreSQL, JPA Entities & Domain Modeling

📥 Input
Day 5 — Full Docker Compose stack running PostgreSQL container healthy on port 5432 Hexagonal package structure in system-api
🔧 Activities
  1. Design domain model — Create Product, Category, Supplier as domain entities in domain.model. Use Java Records for Value Objects. Define relationships: Product belongs to Category, Product has a Supplier.
  2. Create JPA entities — In adapter.out.persistence, create ProductJpaEntity, CategoryJpaEntity, SupplierJpaEntity with @Entity, @Table, @Id, @GeneratedValue(IDENTITY).
  3. Map relationships — @ManyToOne from Product to Category. @ManyToOne from Product to Supplier. Use @JoinColumn. Set fetch = LAZY everywhere (prevent N+1).
  4. Add auditing fields — @CreatedDate, @LastModifiedDate with @EntityListeners(AuditingEntityListener.class). Enable with @EnableJpaAuditing on config.
  5. Configure JPA properties — spring.jpa.hibernate.ddl-auto=validate (Flyway handles schema). spring.jpa.show-sql=true for dev. spring.jpa.open-in-view=false (critical!).
  6. Connect to PostgreSQL — Verify system-api connects to Docker PostgreSQL using compose env vars. Check logs for "HikariPool-1 - Starting..."
  7. Write domain-to-JPA mappers — Static methods or a Mapper class: toDomain(ProductJpaEntity) and toEntity(Product). Keep in persistence adapter package.
📐 Principles
  • SOLID (S) — JPA entity ≠ domain entity. Each has a single responsibility.
  • Clean Architecture — Domain model is persistence-agnostic. No @Entity in domain.
  • OWASP — Parameterized queries via JPA. No string concatenation SQL.
💻 JPA Entity
@Entity
@Table(name = "products")
@EntityListeners(AuditingEntityListener.class)
@Getter @NoArgsConstructor(access = PROTECTED)
public class ProductJpaEntity {

    @Id
    @GeneratedValue(strategy = IDENTITY)
    private Long id;

    @Column(nullable = false, length = 200)
    private String name;

    @Column(nullable = false, precision = 12, scale = 2)
    private BigDecimal price;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false, length = 20)
    private ProductStatus status;

    @ManyToOne(fetch = LAZY)
    @JoinColumn(name = "category_id", nullable = false)
    private CategoryJpaEntity category;

    @ManyToOne(fetch = LAZY)
    @JoinColumn(name = "supplier_id")
    private SupplierJpaEntity supplier;

    @CreatedDate
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;
}

  7. Verify migration — Start system-api. Check logs for "Successfully applied 4 migrations". Query flyway_schema_history table.
  8. Practice rollback strategy — Flyway Community doesn't support undo. Plan forward-only migrations. Write V5__add_column.sql to add a column, then V6__remove_column.sql to remove it.
📐 Principles
  • Infrastructure as Code — Database schema is version-controlled, repeatable, auditable.
  • 12-Factor #1 — One codebase, tracked in revision control. Schema included.
  • Never use ddl-auto=update in prod — Flyway is the single source of truth for schema.
💻 V1__create_categories_table.sql
CREATE TABLE categories (
    id          BIGSERIAL PRIMARY KEY,
    name        VARCHAR(100)  NOT NULL UNIQUE,
    description VARCHAR(500),
    created_at  TIMESTAMP     NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP     NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_categories_name ON categories(name);
💻 V3__create_products_table.sql
CREATE TABLE products (
    id          BIGSERIAL PRIMARY KEY,
    name        VARCHAR(200)    NOT NULL,
    price       NUMERIC(12,2)   NOT NULL CHECK (price >= 0),
    status      VARCHAR(20)     NOT NULL DEFAULT 'DRAFT',
    category_id BIGINT          NOT NULL REFERENCES categories(id),
    supplier_id BIGINT          REFERENCES suppliers(id),
    created_at  TIMESTAMP       NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP       NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_products_category ON products(category_id);
CREATE INDEX idx_products_status   ON products(status);
CREATE INDEX idx_products_name     ON products(name);

💻 REST Controller
@RestController
@RequestMapping("/api/v1/products")
@RequiredArgsConstructor
public class ProductController {

    private final CreateProductUseCase createUC;
    private final GetProductUseCase getUC;

    @PostMapping
    ResponseEntity<ProductResponse> create(
            @Valid @RequestBody CreateProductRequest req) {
        Product p = createUC.create(req.toCommand());
        return ResponseEntity
            .created(URI.create("/api/v1/products/" + p.id()))
            .body(ProductResponse.from(p));
    }

    @GetMapping("/{id}")
    ProductResponse getById(@PathVariable Long id) {
        return ProductResponse.from(getUC.getById(id));
    }

    @GetMapping
    Page<ProductResponse> list(Pageable pageable) {
        return getUC.list(pageable).map(ProductResponse::from);
    }

    @DeleteMapping("/{id}")
    @ResponseStatus(NO_CONTENT)
    void delete(@PathVariable Long id) {
        createUC.delete(id);
    }
}

DAY 9 Phase 2 — System API

Bean Validation, Error Handling & OWASP Hardening

📥 Input
Day 8 — Working CRUD REST API No input validation yet, generic error responses
🔧 Activities
  1. Add Bean Validation annotations — @NotBlank, @NotNull, @Size(min,max), @DecimalMin("0.01"), @Email, @Pattern on all request DTOs. Use @Valid in controller method params.
  2. Create global exception handler — @RestControllerAdvice class. Handle MethodArgumentNotValidException, ConstraintViolationException, EntityNotFoundException, generic Exception.
  3. Implement RFC 7807 Problem Detail — Return ProblemDetail (Spring 6 native) with type, title, status, detail, instance. Content-Type: application/problem+json.
  4. Add field-level error details — For validation errors, return array of {"field":"name","message":"must not be blank"} in the errors extension.
  5. Sanitize inputs — Strip HTML tags from string inputs. Use @SafeHtml or custom validator. Prevent XSS via input sanitization.
  6. Add SQL injection tests — Attempt ' OR 1=1 -- in query params. Verify JPA parameterized queries block it.
  7. Configure security headers — Add X-Content-Type-Options: nosniff, X-Frame-Options: DENY, Strict-Transport-Security via response filter.
  8. Test all error scenarios — Missing required fields, invalid values, non-existent IDs, malformed JSON. Verify consistent error format.
📐 Principles
  • OWASP A03 — Injection prevention via parameterized queries + input validation.
  • OWASP A07 — XSS prevention via input sanitization + security headers.
  • RFC 7807 — Standard error format across all APIs.

💻 Global Exception Handler
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    ProblemDetail handleValidation(
            MethodArgumentNotValidException ex) {
        ProblemDetail pd = ProblemDetail.forStatus(400);
        pd.setTitle("Validation Failed");
        pd.setType(URI.create("/errors/validation"));
        List<Map<String,String>> errors = ex.getFieldErrors()
            .stream()
            .map(e -> Map.of(
                "field", e.getField(),
                "message", e.getDefaultMessage()))
            .toList();
        pd.setProperty("errors", errors);
        return pd;
    }

    @ExceptionHandler(EntityNotFoundException.class)
    ProblemDetail handleNotFound(EntityNotFoundException ex) {
        ProblemDetail pd = ProblemDetail.forStatus(404);
        pd.setTitle("Resource Not Found");
        pd.setDetail(ex.getMessage());
        return pd;
    }
}
