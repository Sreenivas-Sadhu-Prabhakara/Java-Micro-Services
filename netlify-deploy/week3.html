DAY 11 Phase 3 — Middleware

OpenFeign Clients & Inter-Service Communication

📥 Input
Day 10 — System API fully working with Swagger, tests passing Middleware module has hex structure but no business logic yet
🔧 Activities
  1. Add OpenFeign dependency — spring-cloud-starter-openfeign in middleware POM. Add @EnableFeignClients on Application class.
  2. Create Feign client interface — @FeignClient(name="system-api", url="${system-api.url}"). Define methods matching system-api REST endpoints: getProducts(), getProductById(), createProduct().
  3. Configure Feign — Set timeouts in YAML: connectTimeout: 3000, readTimeout: 5000. Enable request/response logging: feign.client.config.default.loggerLevel: FULL.
  4. Create middleware DTOs — Middleware has its own request/response DTOs. Never share DTOs across modules. Map from middleware DTO → Feign request → Feign response → middleware response.
  5. Build middleware service layer — ProductOrchestrationService in application.service. Calls Feign client (outbound adapter), applies middleware business logic (enrichment, transformation).
  6. Create middleware REST endpoints — GET /api/v1/middleware/products. The middleware controller calls the orchestration service, which calls the Feign client to system-api.
  7. Test the chain — BFF:8080 → Middleware:8082 → System-API:8081. Use docker compose up and curl from host through the full chain.
  8. Handle Feign errors — Create FeignErrorDecoder to translate system-api HTTP errors into middleware domain exceptions.
📐 Principles
  • SOLID (D) — Feign client is an outbound adapter. Service depends on port interface, not Feign directly.
  • Loose Coupling — Middleware never directly imports system-api classes. Communication via HTTP/JSON only.
  • Timeout First — Always configure connect and read timeouts. Never rely on defaults (infinite).

DAY 12 Phase 3 — Middleware

Service Composition & Business Orchestration

📥 Input
Day 11 — OpenFeign client calling system-api Basic enrichment logic in middleware
🔧 Activities
  1. Compose multiple downstream calls — Fetch product + category + supplier from system-api in a single middleware endpoint. Aggregate into a rich response: ProductDetailView.
  2. Implement parallel calls — Use CompletableFuture.supplyAsync() to call product, category, and supplier endpoints in parallel. Combine results with allOf().
  3. Add business rules — Calculate tax, apply discounts based on category, check inventory thresholds, generate display labels. This is the middleware's core value.
  4. Create aggregation endpoints — GET /api/v1/catalog/overview returns top products, category stats, recent additions — all composed from multiple system-api calls.
  5. Implement DTO mapping layer — Create mapper classes (or use MapStruct) to transform between Feign response DTOs, domain models, and middleware response DTOs.
  6. Handle partial failures — If supplier call fails but product succeeds, return product with supplier: null rather than failing the entire request. Graceful degradation.
  7. Add request correlation IDs — Generate UUID on first request. Pass as X-Correlation-ID header through Feign. Log it at every layer for traceability.
📐 Principles
  • Saga Pattern (simple) — Compose multi-step operations with graceful partial failure handling.
  • CQRS-lite — Read (query) endpoints aggregate data. Write (command) endpoints pass through to system-api.
  • Correlation — Every request traceable across services via X-Correlation-ID.
💻 Parallel Composition
@Service
@RequiredArgsConstructor
public class CatalogOrchestrationService {

    private final SystemApiClient systemApi;

    public ProductDetailView getProductDetail(Long id) {
        CompletableFuture<ProductDTO> productF =
            supplyAsync(() -> systemApi.getProductById(id));
        CompletableFuture<CategoryDTO> categoryF =
            supplyAsync(() -> systemApi.getCategoryById(
                productF.join().categoryId()));

        return ProductDetailView.builder()
            .product(productF.join())
            .category(categoryF.join())
            .tax(calculateTax(productF.join().price()))
            .build();
    }

    public CatalogOverview getOverview() {
        var topProducts = supplyAsync(
            () -> systemApi.getProducts(0, 10));
        var categories = supplyAsync(
            () -> systemApi.getCategories());
        CompletableFuture.allOf(topProducts, categories).join();

        return CatalogOverview.builder()
            .featured(topProducts.join().getContent())
            .categoryCount(categories.join().size())
            .build();
    }
}
💻 Correlation ID Interceptor
@Component
public class CorrelationIdInterceptor
        implements RequestInterceptor {

    public static final String HEADER = "X-Correlation-ID";

    @Override
    public void apply(RequestTemplate template) {
        String corrId = MDC.get("correlationId");
        if (corrId == null) corrId = UUID.randomUUID().toString();
        template.header(HEADER, corrId);
    }
}

DAY 13 Phase 3 — Middleware

Resilience4j — Circuit Breakers, Retries & Bulkheads

📥 Input
Day 12 — Middleware orchestrating system-api calls No fault tolerance — if system-api is down, middleware crashes
🔧 Activities
  1. Add Resilience4j dependency — resilience4j-spring-boot3 + resilience4j-feign. Integrates directly with Spring Boot auto-config.
  2. Configure circuit breaker — Sliding window of 10 calls. Failure rate threshold 50%. Open state duration 30s. Half-open allows 3 probe calls.
  3. Add @CircuitBreaker on service methods — @CircuitBreaker(name="systemApi", fallbackMethod="fallback"). Write fallback methods returning cached/default data.
  4. Configure retry — Max 3 attempts with exponential backoff (500ms, 1s, 2s). Retry only on FeignException.ServiceUnavailable and IOException.
  5. Configure bulkhead — Limit concurrent calls to system-api to 20 (semaphore bulkhead). Prevent thread exhaustion under load.
  6. Configure time limiter — 5s timeout per call. If system-api is slow, fail fast rather than blocking threads.
  7. Test circuit breaker — Stop system-api container. Call middleware. Verify circuit opens after threshold. See fallback responses. Restart system-api. Verify circuit closes.
  8. Add Resilience4j Actuator — Enable /actuator/circuitbreakers and /actuator/retries endpoints. Monitor circuit state in real-time.
📐 Principles
  • Design for Failure — Every remote call can fail. Plan for it, don't hope it won't.
  • Bulkhead Pattern — Isolate failures. One slow dependency shouldn't crash all threads.
  • Graceful Degradation — Return stale/cached/default data rather than errors.
💻 Circuit Breaker + Fallback
@Service
@RequiredArgsConstructor
public class ResilientProductService {

    private final SystemApiClient systemApi;

    @CircuitBreaker(name = "systemApi",
                    fallbackMethod = "fallbackProducts")
    @Retry(name = "systemApi")
    @Bulkhead(name = "systemApi")
    @TimeLimiter(name = "systemApi")
    public CompletableFuture<List<ProductDTO>> getProducts() {
        return supplyAsync(
            () -> systemApi.getProducts(0, 20).getContent());
    }

    // Fallback: return cached/empty data
    private CompletableFuture<List<ProductDTO>> fallbackProducts(
            Throwable t) {
        log.warn("Circuit open, returning fallback: {}",
                 t.getMessage());
        return completedFuture(Collections.emptyList());
    }
}

DAY 14 Phase 3 — Middleware

Global Error Handling (RFC 7807) & Middleware Tests

📥 Input
Day 13 — Resilience4j patterns configured Middleware has no unified error handling or tests
🔧 Activities
  1. Create middleware @RestControllerAdvice — Handle FeignException (downstream errors), CallNotPermittedException (circuit open), BulkheadFullException, RequestNotPermitted.
  2. Map downstream errors — system-api 400 → middleware 400 (pass through). system-api 404 → middleware 404. system-api 500 → middleware 502 (Bad Gateway). Circuit open → 503 (Service Unavailable).
  3. Consistent RFC 7807 across middleware — Same ProblemDetail format as system-api. Include X-Correlation-ID in error responses for traceability.
  4. Write unit tests for orchestration service — Mock Feign client with Mockito. Test happy path, partial failures, full failures. Verify fallback behavior.
  5. Write circuit breaker integration tests — Use WireMock to simulate system-api. Return 500s to trigger circuit open. Verify fallback. Return 200s to close circuit.
  6. Write controller tests — MockMvc tests for middleware endpoints. Verify response structure, status codes, correlation IDs.
  7. Add logging standards — Structured JSON logging with logback-spring.xml. Fields: timestamp, level, correlationId, service, message. MDC for correlation.
📐 Principles
  • Error Translation — Never expose downstream stack traces. Map to appropriate HTTP codes.
  • WireMock — Simulate downstream services in tests. No dependency on running system-api.
  • Structured Logging — JSON logs for machine parsing. MDC for distributed tracing context.
💻 Middleware Exception Handler
@RestControllerAdvice
@Slf4j
public class MiddlewareExceptionHandler {

    @ExceptionHandler(FeignException.class)
    ProblemDetail handleFeign(FeignException ex) {
        int status = switch (ex.status()) {
            case 400 -> 400;
            case 404 -> 404;
            default  -> 502; // Bad Gateway
        };
        ProblemDetail pd = ProblemDetail.forStatus(status);
        pd.setTitle("Downstream Service Error");
        pd.setProperty("correlationId", MDC.get("correlationId"));
        return pd;
    }

    @ExceptionHandler(CallNotPermittedException.class)
    ProblemDetail handleCircuitOpen(CallNotPermittedException ex) {
        ProblemDetail pd = ProblemDetail.forStatus(503);
        pd.setTitle("Service Temporarily Unavailable");
        pd.setDetail("Circuit breaker is OPEN for: " +
                      ex.getCausingCircuitBreakerName());
        return pd;
    }
}

💻 MapStruct Mapper
@Mapper(componentModel = "spring")
public interface ProductMapper {

    @Mapping(target = "displayName",
             expression = "java(product.name().toUpperCase())")
    @Mapping(target = "formattedPrice",
             expression = "java(formatCurrency(product.price()))")
    EnrichedProductResponse toEnrichedResponse(Product product);

    default String formatCurrency(BigDecimal price) {
        return NumberFormat.getCurrencyInstance(Locale.US)
                           .format(price);
    }
}
💻 Response Envelope
public record ApiResponse<T>(
    T data,
    ResponseMetadata metadata
) {
    public static <T> ApiResponse<T> of(T data) {
        return new ApiResponse<>(data,
            new ResponseMetadata(
                MDC.get("correlationId"),
                Instant.now()));
    }
}

public record ResponseMetadata(
    String correlationId,
    Instant timestamp
) {}
